This Blueprint function calculates the player‚Äôs Dynamic Attack Rate by iterating through stored combat data for each enemy. It performs multiple summations ‚Äî including weighted attack impact (WiHi), penalty modifiers (PjMj), and temporal decay factors (T and G) ‚Äî to produce a balanced metric that reflects real-time combat performance. The system integrates time-based decay using exponential functions and aggregates results to determine the final Attack Rate Output, influencing player abilities and special attack activation during gameplay.

This section focuses on calculating the summation of Wi, representing the total contribution of each successful attack weighted by importance. It retrieves the enemy‚Äôs stored attack data, loops through each event, and adds values adjusted by an exponential decay based on time, ensuring older actions have less influence.

This part computes two core parameters: WiHi: The total weighted impact of all successful hits. PjMj: The accumulated penalty for missed or failed attacks. Both are processed with time-based decay and stored for the final calculation, ensuring the player‚Äôs consistency and precision directly affect the attack rate.

This section handles temporal and gameplay gap adjustments. The summation of T accounts for the average attack timing intervals, while G integrates global gameplay modifiers such as overall combo flow or gap duration. These values stabilize the formula to maintain fairness and responsiveness.

The last part of the Blueprint compiles all the computed summations ‚Äî WiHi, PjMj, T, and G ‚Äî into the final Attack Rate formula. It divides the total weighted hit values by the combined penalties and timing modifiers to generate a single normalized output. This output dynamically controls combat performance metrics and unlocks special abilities when thresholds are met.

This setup defines and stores all combat-related information for each enemy. The EnemyAttackData structure contains arrays for attacks, hits, penalties, and hit checks, along with a combo boolean to track combo states. These values are organized in an Enemy Data Map, allowing the system to record and update individual enemy interactions dynamically. This structure ensures accurate calculation of the player‚Äôs attack rate and performance per enemy during combat.

This blueprint visualizes the logic for handling enemy damage in the combat system. When the player attacks, the system detects a hit using a sphere trace, identifies the enemy, and applies calculated damage based on player stats and attack type. The setup ensures accurate damage registration, organized data management, and smooth interaction between player and enemy systems.

This blueprint represents the Finisher Attack system created inside the BP_ThirdPersonCharacter Event Graph. Overall, this system integrates player movement, targeting, and combat states into a cohesive and responsive finishing move sequence.
